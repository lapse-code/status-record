## MVP 功能完成条件

### 计时器

- 用户可以启动默认番茄钟。当前已实现。
- 用户可以输入自定义倒计时。当前已实现。
- 用户可以暂停、继续、取消。暂停会记录当前真实专注片段并进入拖延；继续仍回到同一轮；取消不记录本轮专注。当前已实现并由单元测试和 E2E 覆盖。
- 倒计时结束后进入复盘。当前已实现。
- 刷新页面后能恢复合理状态。当前通过 IndexedDB 时间戳和暂停字段支持，仍建议继续手动试用。

### 到岗和拖延

- 用户可以点击到岗。当前已实现。
- 用户未到岗时直接开始番茄钟，会自动创建到岗记录并绑定本轮专注。当前已实现并由 E2E 覆盖。
- 用户已到岗时开始番茄钟，必须复用已有到岗记录，不得重置到岗时间；如果历史数据里已有重复未关闭到岗，系统以最早那条作为有效到岗。当前已实现并由 E2E 覆盖。
- 系统记录到岗时间。当前已实现。
- 第一轮专注开始后自动计算拖延。当前已实现。
- 未开始专注的到岗记录不会被错误统计为 0 拖延。当前已由单元测试覆盖。

### 复盘

- 每轮完成后可以选择状态。当前已实现。
- 可以输入注意力切换次数。当前已实现。
- 可以选择产物标签并写文字。当前已实现。
- 可以选择不专注原因标签，支持“无”。当前已实现。
- 可以选择立即使用或保存休息余额。当前已实现。
- 复盘提交后记录进入统计。当前已实现并由 E2E 覆盖。

### 休息余额和休息倒计时

- 今日已完成番茄钟真实有效分钟每累计满 25 分钟获得 5 分钟休息，可跨多轮累计。当前已实现并有单元测试和 E2E 覆盖。
- 自定义时长、提前完成和完整完成都按真实有效分钟写入今日学习累计账本。当前已实现。
- 取消本轮不会写入今日学习累计账本，不产生休息余额。当前已实现。
- 今日休息余额和未满 25 分钟的累计进度不跨天保存。当前已实现并有单元测试。
- 使用休息会减少今日余额。当前已实现。
- 保存休息不会丢失今日余额，但未用余额第二天清零。当前已实现。
- 复盘中选择“使用休息”会启动休息倒计时。当前已实现并由 E2E 覆盖。
- 休息倒计时可以提前结束，未用完分钟退回余额。当前已实现并由单元测试和 E2E 覆盖。
- 休息自然结束后，如果仍有余额会弹窗让用户选择继续休息或开始专注；如果没有余额则自动进入下一轮拖延记录。当前已实现并由 E2E 覆盖。

### 睡眠

- 每天可以记录睡眠时长。当前已实现。
- 每天可以记录主观精力 1-5。当前已实现。
- 同一天重复记录会修改旧记录。当前已实现。

### 标签

- 默认标签初始化。当前已实现。
- 用户可以新增标签。当前已实现。
- 用户可以隐藏或停用标签。当前已实现。
- 历史记录中的旧标签仍可统计。当前 schema 支持。

### 统计

- 可以查看日、周、月专注时长。当前已实现。
- 可以查看日、周、月拖延。当前已实现。
- 可以查看一天的 5 分钟点阵，颜色区分拖延、休息、专注、不专注，并支持前一天、后一天、今天和日期输入查看历史。当前已实现并由单元测试和 E2E 覆盖。
- 点阵会实时显示打开到岗、运行中专注、运行中休息、暂停拖延和完成未复盘状态；取消本轮后临时专注颜色会撤销。当前已实现并由单元测试覆盖。
- 点阵按记录发生时区绘制，历史记录不会因为当前设备时区变化而漂移；旧数据缺少时区时按 `Asia/Tokyo` 兼容。当前已实现并由单元测试覆盖。
- 可以查看一周的 5 分钟点阵，按周一到周日展示 7 天，并支持上一周、下一周、本周和日期跳转。当前已实现并由 E2E 覆盖。
- 可以查看注意力切换次数和每小时切换次数。当前已实现。
- 可以查看状态分布。当前已实现。
- 可以查看产物标签分布和复盘记录明细。当前已实现。
- 可以查看不专注原因分布，图表排除默认“无”；没有原因时显示空状态。当前已实现并由 E2E 覆盖。
- 可以查看睡眠时长和精力趋势。当前已实现。
- 可以点击状态、产物、不专注原因饼图扇区或右侧标签，筛选下方“记录明细”。当前已实现并由 E2E 覆盖。

### 数据

- 数据本地持久化。当前已实现。
- 支持 JSON 导出。当前已实现，格式为 `status-record.backup` version 1。
- 支持 JSON 导入。当前已实现，支持 version 1 备份格式并兼容早期 `AppSnapshot` 导出结构；导入为合并写入，不会先清空本地数据。
- 支持加载 2026-06-01 到 2026-06-10 示例数据，不删除真实记录。当前已实现并由 E2E 覆盖。
- 示例数据、导入、导出入口在侧栏隐藏时仍能从主内容区域访问。当前已实现并由 E2E 覆盖。
- 统计结果可从源记录重算。当前已实现。
- schema 变更有 migration。当前 Dexie schema 已升级到 version 4，用于支持 `break_sessions`、`focus_segments` 和源记录 `time_zone`；后续变更需要继续新增 migration。
