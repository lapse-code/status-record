## MVP 功能完成条件

### 计时器

- 用户可以启动默认番茄钟。当前已实现。
- 用户可以输入自定义倒计时。当前已实现。
- 用户可以暂停、继续、取消。当前已实现。
- 倒计时结束后进入复盘。当前已实现。
- 刷新页面后能恢复合理状态。当前通过 IndexedDB 时间戳和暂停字段支持，仍建议继续手动试用。

### 到岗和启动延迟

- 用户可以点击到岗。当前已实现。
- 系统记录到岗时间。当前已实现。
- 第一轮学习开始后自动计算启动延迟。当前已实现。
- 未开始学习的到岗记录不会被错误统计为 0 延迟。当前已由单元测试覆盖。

### 复盘

- 每轮完成后可以选择状态。当前已实现。
- 可以输入注意力切换次数。当前已实现。
- 可以选择产物标签并写文字。当前已实现。
- 可以选择阻塞标签，支持“无”。当前已实现。
- 可以选择立即使用或保存休息余额。当前已实现。
- 复盘提交后记录进入统计。当前已实现并由 E2E 覆盖。

### 休息余额和休息倒计时

- 每完成 25 分钟学习获得 5 分钟休息。当前已实现并有单元测试。
- 自定义时长按规则获得休息。当前已实现。
- 使用休息会减少余额。当前已实现。
- 保存休息不会丢失余额。当前已实现。
- 复盘中选择“使用休息”会启动休息倒计时。当前已实现并由 E2E 覆盖。
- 休息倒计时可以提前结束，未用完分钟退回余额。当前已实现并由单元测试和 E2E 覆盖。
- 休息结束后自动进入下一轮启动延迟记录，用户可以选择下一轮番茄钟时间继续。当前已实现。

### 睡眠

- 每天可以记录睡眠时长。当前已实现。
- 每天可以记录主观精力 1-5。当前已实现。
- 同一天重复记录会修改旧记录。当前已实现。

### 标签

- 默认标签初始化。当前已实现。
- 用户可以新增标签。当前已实现。
- 用户可以隐藏或停用标签。当前已实现。
- 历史记录中的旧标签仍可统计。当前 schema 支持。

### 统计

- 可以查看日、周、月学习时长。当前已实现。
- 可以查看日、周、月启动延迟。当前已实现。
- 可以查看一天的 5 分钟点阵，颜色区分启动延迟、正常学习、阻塞/被打断，并支持前一天、后一天、今天和日期输入查看历史。当前已实现并由单元测试和 E2E 覆盖。
- 可以查看注意力切换次数和每小时切换次数。当前已实现。
- 可以查看状态分布。当前已实现。
- 可以查看产物标签分布和复盘记录明细。当前已实现。
- 可以查看阻塞原因分布。当前已实现。
- 可以查看睡眠时长和精力趋势。当前已实现。
- 可以点击状态、产物、阻塞饼图扇区或右侧标签，筛选下方“记录明细”。当前已实现并由 E2E 覆盖。

### 数据

- 数据本地持久化。当前已实现。
- 支持 JSON 导出。当前已实现。
- 支持加载 2026-06-01 到 2026-06-10 示例数据，不删除真实记录。当前已实现并由 E2E 覆盖。
- 统计结果可从源记录重算。当前已实现。
- schema 变更有 migration。当前 Dexie schema 已升级到 version 2，用于支持 `break_sessions`；后续变更需要继续新增 migration。

## 工程完成条件

- TypeScript 无类型错误。当前 `npm run build` 已通过。
- 核心 domain 逻辑有单元测试。当前 `npm run test:run` 已通过。
- 主流程有至少一条 E2E 测试。当前 `npm run e2e` 已通过。
- 代码格式化和 lint 通过。当前 `npm run lint` 已通过。
- 没有明显无用依赖。当前依赖均用于功能或测试。
- README 或 docs 能指导本地启动。当前已实现。

## 发布前手动检查

- 新用户首次打开能直接开始使用。
- 完成一轮 25 分钟番茄钟后数据正确。
- 完成一轮自定义番茄钟后数据正确。
- 休息余额计算正确。
- 日/周/月统计口径一致。
- 状态、产物、阻塞标签筛选能正确切换“记录明细”。
- 睡眠记录可新增和修改。
- 标签新增后能在复盘和统计中出现。
