# Testing Strategy

## 测试目标

测试重点是保证时间、状态、统计口径可靠。这个工具的价值来自记录可信，如果计时和统计错了，界面再好也没有意义。

## 单元测试

优先覆盖纯函数：

- 休息余额计算：
  - 25 分钟获得 5 分钟。
  - 50 分钟获得 10 分钟。
  - 24 分钟获得 0 分钟。
  - 15 + 15 分钟跨轮累计后获得 5 分钟，并留下 5 / 25 分钟进度。
  - 今日休息余额和未满 25 分钟的进度不跨天保留。
  - 取消本轮不进入今日学习累计账本。
  - 提前结束休息时，只扣实际使用分钟，未用完分钟退回余额。
- 兼容启动延迟工具：
  - 到岗后 30 分钟开始，延迟 30。
  - 到岗后未开始，状态为 `not_started`。
  - 多个 focus sessions 时只使用第一轮。
- 拖延统计：
  - UI 中“今日拖延”和统计页 trend 必须等于日点阵 `startup_delay` 真实时长总和。
  - 完成一轮后仍在岗等待下一轮的时间应继续进入拖延统计。
- 日/周/月聚合：
  - 专注时长累计。
  - 周/月统计使用轻量区间汇总，不应为了聚合范围生成完整日点阵。
  - 轻量区间汇总必须和日点阵的 `durationMsByState` 使用同一套时间覆盖优先级。
  - 注意力切换次数累计。
  - 标签频次统计。
  - 睡眠均值。
- 日点阵：
  - 一天固定 288 个 5 分钟点。
  - 每个 5 分钟点按源记录与 cell 的秒级真实重叠时长决定主色，并保存每种状态的真实时长。
  - 1 分钟拖延 + 4 分钟专注时显示专注；4 分钟拖延 + 1 分钟专注时显示拖延。
  - 跨过 5 分钟边界的 5 分钟休息，真实休息总时长仍应为 5 分钟；UI 用混合分段表示，不允许通过删除时间来限制蓝点数量。
  - 暂停会把当前专注片段落账，暂停区间显示为拖延，继续后新片段显示为专注。
  - 打开的到岗记录会用当前 `now` 实时显示拖延。
  - 运行中的专注片段会用当前 `now` 实时显示专注；完成但未复盘时先显示专注。
  - 取消的专注片段不会保留为专注；若到岗仍打开，取消后的时间继续显示拖延。
  - 运行中的休息倒计时会用当前 `now` 实时显示休息。
  - 状态时长打平时按黄色不专注、绿色专注、红色拖延、天蓝色休息的优先级决定颜色。
  - 拖延点为红色口径。
  - 专注点为绿色口径。
  - 不专注点为黄色口径。
  - 休息点为天蓝色口径。
- 时区处理：
  - 同一个 UTC 时间戳在东京和洛杉矶会映射到不同 `local_date`。
  - 历史记录按自身 `time_zone` 放入日点阵对应本地分钟，不受当前设备时区影响。
  - 跨午夜的专注片段会在开始日末尾和次日开头分别显示。
  - 旧数据或旧备份缺少 `time_zone` 时按 `Asia/Tokyo` 回填。
- 标签规则：
  - “无”不专注原因不能和其他不专注原因并选。
- 归档标签不影响历史统计，但不再出现在新的复盘选择中。
- 已有历史记录的标签不能删除，只能归档。
- `status-completed` 不能归档或删除，且是唯一让已复盘专注片段保持绿色的状态；不专注原因标签不参与点阵颜色判定。
- 点阵颜色设置保存后，应影响日点阵、周点阵和图例展示。

- `src/domain/analytics.test.ts`
- `src/domain/time.test.ts`
- Analytics 单元测试覆盖混合拖延、休息、不专注场景下，范围汇总时长与日点阵真实时长保持一致。

## 组件测试

覆盖核心 UI：

- Today 页面显示当前到岗状态和计时器。
- 自定义时间输入校验。
- 倒计时结束后弹出复盘。
- 复盘表单必填状态。
- 注意力切换次数不能为负数。
- 睡眠一天记录一次，再次输入会修改。

## 集成测试

覆盖跨模块流程：

1. 到岗。
2. 等待或模拟 30 分钟。
3. 开始 25 分钟番茄钟。
4. 完成番茄钟。
5. 提交复盘。
6. 验证：
   - 专注时长增加 25。
   - 拖延为 30。
   - 休息余额增加 5。
   - 状态、产物、不专注原因标签进入统计。

## E2E 测试

使用 Playwright 覆盖用户主路径：

- 首次打开应用，默认标签已初始化。
- 完成一轮番茄钟并提交复盘。
- 记录睡眠。
- 查看 Analytics 页面。
- 刷新页面后数据仍存在。
- 导出 JSON 成功。
- 导入 JSON 备份后，统计页能读取导入记录。

当前已有 E2E：

- `tests/e2e/main-flow.spec.ts`
- 覆盖到岗、开始自定义番茄钟、手动完成、填写复盘、保存进入统计。
- 覆盖已有到岗时开始番茄钟不会新增或重置到岗记录；覆盖历史数据存在重复未关闭到岗时，开始番茄钟绑定最早打开的到岗记录。
- 覆盖复盘保存、使用休息、提前结束休息都不会关闭、重开或重置当前到岗记录；只有点击离开会写入 `left_at`。
- 覆盖今日 15 + 15 分钟跨轮累计后获得 5 分钟休息，并可立即启动休息倒计时。
- 覆盖刷新后恢复正在运行的倒计时。
- 覆盖复盘后选择使用休息、进入休息倒计时、提前结束休息并保持原到岗记录打开。
- 覆盖休息自然结束后仍有余额时弹出继续休息选择，并可继续休息 5 分钟。
- 覆盖暂停后显示“暂停中，正在记录拖延”，继续后回到同一轮专注，完成后进入复盘。
- 覆盖每日睡眠记录 upsert，并验证默认 `7:00`、15 分钟步进、键盘输入和精力步进。
- 覆盖新增和归档自定义标签。
- 覆盖专注绑定状态标签不能归档或删除。
- 覆盖加载 2026-06-01 到 2026-06-10 示例数据，并验证统计页可见。
- 覆盖统计页月视图和周视图只显示范围聚合统计，不显示日点阵和记录明细；覆盖日/周/月日期控制统一在统计顶部；覆盖时间占比显示专注、不专注、拖延且休息不进入分母。
- 覆盖统计页日视图下日点阵可见，并能通过前一天/后一天切换日期；断言兼容宽屏每列 30 分钟和窄屏每列 1 小时两种展示。
- 覆盖周点阵页面可见，支持上一周、下一周和选择日期跳到所在周。
- 覆盖统计页日视图点击状态、产物、不专注原因图例和饼图扇区后，“记录明细”按对应标签筛选。
- 覆盖统计页日视图编辑已有记录明细，保存后明细内容刷新，且编辑流程不涉及休息余额。
- 覆盖导入 `status-record.backup` JSON 后，本地记录和统计页出现导入的复盘内容。
- 覆盖只有“不专注原因 = 无”的导入记录不会在不专注原因图表中生成“无”扇区，而是显示“暂无不专注原因记录”。
- 覆盖侧栏操作区被隐藏时，主内容顶部仍显示“示例数据 / 导入 / 导出”，并且导入功能可用。
- Playwright 配置使用本机 Google Chrome channel，避免必须下载 Playwright 自带 Chromium。
- E2E 会固定浏览器时间为 `2026-06-12T12:00:00+09:00`，避免真实机器跨过午夜后影响“今日休息余额不跨天”的断言。

## 时间测试

- 使用 fake timers，不依赖真实等待。
- 测试跨日场景：
  - 晚上到岗，凌晨完成。
  - 睡眠记录归属到用户选择的日期。
- 测试跨时区场景：
  - 东京记录在东京日点阵中保持原来的小时位置。
  - 切换到其他设备时区后，已有记录不重新漂移。
- 测试跨午夜场景：
  - 同一段专注如果跨过记录时区的 00:00，应在两天点阵中分别显示。
- 测试浏览器刷新恢复：
  - running timer。
  - completed but not reviewed session。

## 手动验收

每个版本发布前至少手动检查：

- 一轮 25 分钟流程。
- 一轮自定义时长流程。
- 结束复盘。
- 休息余额使用和保存。
- 休息倒计时自然结束和提前结束。
- 休息结束后如果仍有余额，继续休息弹窗提供 5 分钟、全部余额和自定义分钟选项。
- 番茄钟自然结束和休息自然结束时浏览器通知、默认提示音、页面内提示均可触发。
- 睡眠记录修改。
- 日/周/月统计切换。
- 统计页半屏或窄屏布局下，分布图不应出现饼图和标签重叠。
- 数据导出文件可读。
- 数据导入文件能恢复到统计页。
- 在窄屏、半屏或其他会隐藏侧栏操作区的布局下，示例数据、导入、导出入口仍可见可用。

## 后续质量指标

- 核心 domain 逻辑单元测试覆盖率保持在较高水平。
- 对 UI 覆盖率不追求数字，优先覆盖真实风险路径。
- 每个 schema migration 都需要迁移测试。
