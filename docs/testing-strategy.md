# Testing Strategy

## 测试目标

测试重点是保证时间、状态、统计口径可靠。这个工具的价值来自记录可信，如果计时和统计错了，界面再好也没有意义。

## 单元测试

优先覆盖纯函数：

- 休息余额计算：
  - 25 分钟获得 5 分钟。
  - 50 分钟获得 10 分钟。
  - 24 分钟获得 0 分钟。
  - 自定义时长使用 `floor(minutes / 25) * 5`。
  - 提前结束休息时，只扣实际使用分钟，未用完分钟退回余额。
- 启动延迟计算：
  - 到岗后 30 分钟开始，延迟 30。
  - 到岗后未开始，状态为 `not_started`。
  - 多个 focus sessions 时只使用第一轮。
- 日/周/月聚合：
  - 学习时长累计。
  - 注意力切换次数累计。
  - 标签频次统计。
  - 睡眠均值。
- 日点阵：
  - 一天固定 288 个 5 分钟点。
  - 启动延迟点为红色口径。
  - 正常学习点为绿色口径。
  - 有阻塞或被打断的学习点为黄色口径。
- 标签规则：
  - “无”阻塞不能和其他阻塞并选。
  - 隐藏标签不影响历史统计。

当前已有单元测试：

- `src/domain/break-bank.test.ts`
- `src/domain/startup-delay.test.ts`
- `src/domain/analytics.test.ts`

## 组件测试

覆盖核心 UI：

- Today 页面显示当前到岗状态和计时器。
- 自定义时间输入校验。
- 倒计时结束后弹出复盘。
- 复盘表单必填状态。
- 注意力切换次数不能为负数。
- 睡眠一天记录一次，再次输入会修改。

## 集成测试

覆盖跨模块流程：

1. 到岗。
2. 等待或模拟 30 分钟。
3. 开始 25 分钟番茄钟。
4. 完成番茄钟。
5. 提交复盘。
6. 验证：
   - 学习时长增加 25。
   - 启动延迟为 30。
   - 休息余额增加 5。
   - 状态、产物、阻塞标签进入统计。

## E2E 测试

使用 Playwright 覆盖用户主路径：

- 首次打开应用，默认标签已初始化。
- 完成一轮番茄钟并提交复盘。
- 记录睡眠。
- 查看 Analytics 页面。
- 刷新页面后数据仍存在。
- 导出 JSON 成功。

当前已有 E2E：

- `tests/e2e/main-flow.spec.ts`
- 覆盖到岗、开始自定义番茄钟、手动完成、填写复盘、保存进入统计。
- 覆盖刷新后恢复正在运行的倒计时。
- 覆盖复盘后选择使用休息、进入休息倒计时、提前结束休息并回到下一轮启动延迟。
- 覆盖每日睡眠记录 upsert，并验证默认 `7:00`、15 分钟步进、键盘输入和精力步进。
- 覆盖新增和隐藏自定义标签。
- 覆盖加载 2026-06-01 到 2026-06-10 示例数据，并验证统计页可见。
- 覆盖示例数据下日点阵可见，并能通过前一天/后一天切换日期。
- 覆盖点击状态、产物、阻塞图例和饼图扇区后，“记录明细”按对应标签筛选。
- Playwright 配置使用本机 Google Chrome channel，避免必须下载 Playwright 自带 Chromium。

## 时间测试

- 使用 fake timers，不依赖真实等待。
- 测试跨日场景：
  - 晚上到岗，凌晨完成。
  - 睡眠记录归属到用户选择的日期。
- 测试浏览器刷新恢复：
  - running timer。
  - completed but not reviewed session。

## 手动验收

每个版本发布前至少手动检查：

- 一轮 25 分钟流程。
- 一轮自定义时长流程。
- 结束复盘。
- 休息余额使用和保存。
- 休息倒计时自然结束和提前结束。
- 睡眠记录修改。
- 日/周/月统计切换。
- 数据导出文件可读。

## 当前验证命令

```bash
npm run test:run
npm run lint
npm run build
npm run e2e
```

## 后续质量指标

- 核心 domain 逻辑单元测试覆盖率保持在较高水平。
- 对 UI 覆盖率不追求数字，优先覆盖真实风险路径。
- 每个 schema migration 都需要迁移测试。
