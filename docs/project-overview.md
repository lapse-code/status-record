# Project Overview

## 产品定位

Status Record 是一个面向个人学习和深度工作的网页工具。第一版目标不是做复杂的任务管理，而是用一个低摩擦的番茄钟流程，把每天真正发生的学习、拖延、注意力切换、产出、阻塞和睡眠状态记录下来，并能按日、周、月复盘。

## 核心问题

用户希望避免主观美化记录，尤其是启动延迟和学习时长。因此系统应尽量使用计时器和时间戳生成客观数据，只在必须人工判断的地方使用标签和短文本。

## 第一版用户流程

1. 用户到工位或学习位置后，点击“到岗/开始记录”。
2. 系统开始记录到岗时间，但不算学习时长。
3. 用户真正开始学习时，启动固定或自定义番茄钟。
4. 番茄钟结束后，系统弹出复盘表单：
   - 选择本轮状态。
   - 输入注意力切换次数。
   - 记录可见产物标签和文字。
   - 选择主要阻塞标签。
   - 选择是否使用累计休息时间。
5. 系统把本轮学习时长、启动延迟、注意力切换、产物、阻塞和状态写入记录。
6. 用户每天记录一次睡眠时长和主观精力，之后可修改。
7. 用户查看日、周、月统计。

## 每天记录的指标

| 指标 | MVP 记录方式 | 统计方式 |
| --- | --- | --- |
| 学习时长 | 根据完成的番茄钟或自定义计时器自动累计 | 日、周、月总分钟数，按状态/标签拆分 |
| 启动延迟 | 从到岗时间到当天第一轮番茄钟开始时间自动计算 | 日、周、月总分钟数和平均值 |
| 注意力切换次数 | 每轮结束后输入一个数字 | 日、周、月总次数和每小时次数 |
| 可见产物 | 标签 + 短文本 | 标签频次、产物列表、按时间回顾 |
| 主要阻塞 | 标签选择，支持“无”和自定义标签 | 标签频次、趋势、与学习时长的关系 |
| 睡眠 | 每天一次：15 分钟步进的睡眠时长 + 主观精力 1-5 | 日记录、周/月均值、与学习表现对照 |

## 倒计时结束后的五种状态

五种状态应作为可扩展标签保存，MVP 可以提供以下默认值：

| 状态 | 含义 |
| --- | --- |
| 完成 | 本轮基本按计划完成 |
| 部分完成 | 做了，但没有达到预期范围 |
| 被打断 | 外部事件导致中断或效果下降 |
| 卡住 | 主要时间花在理解、调试、搜索或犹豫上 |
| 放弃/转移 | 本轮没有继续原计划，改做别的或停止 |

后续如果用户觉得这五类不准确，应允许在设置里改名、隐藏或新增。

## 休息时间规则

- 每完成 25 分钟学习，系统自动给休息余额增加 5 分钟。
- 自定义倒计时按 `floor(完成学习分钟 / 25) * 5` 计算新增休息余额。
- 每轮结束后询问用户是否现在使用休息时间。
- 用户可以选择：
  - 使用一部分或全部休息余额，并启动休息倒计时。
  - 保留休息余额，直接进入下一轮启动延迟记录。
- 休息倒计时可以提前结束；提前结束时，未用完的分钟退回休息余额。
- 休息结束后，系统重新开始记录下一轮启动延迟，用户再选择番茄钟时长继续。

## MVP 范围

- 网页应用。
- 本地优先存储。
- 固定倒计时和自定义倒计时。
- 到岗记录和启动延迟自动计算。
- 每轮结束复盘。
- 休息倒计时和休息结束提醒。
- 标签管理。
- 睡眠每日记录。
- 日、周、月统计。
- 一天 5 分钟粒度的点阵视图。
- 数据导出/导入，至少支持 JSON，后续可加 CSV。

## 暂不做

- 多人协作。
- 云同步。
- 复杂任务管理。
- 自动检测手机或应用切换。
- 精确自动测量深度工作分钟。
- Mac/iOS 原生壳，等网页版本稳定后再评估。
